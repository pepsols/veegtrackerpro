# 🤖 AI Deployment Manifest - Veegtracker Pro

Deze samenvatting is bedoeld voor de AI om direct acties uit te voeren voor deployment en beheer.

## 📋 Project Context
- **Naam:** Veegtracker Pro
- **Platform:** Android (Kotlin/Compose) + Web (HTML/Firebase JS SDK)
- **Cloud Stack:** Firebase (Auth, Firestore, Hosting, App Distribution)
- **Web Assets Map:** `app/src/main/assets/web/`

## 🛠️ AI Deployment Commando's

### 1. Web Dashboard (Meldkamer) Deployen
Gebruik dit om de HTML-interface live te zetten op Firebase Hosting:
```bash
firebase deploy --only hosting
```

### 2. Test Versie (APK) naar Telefoons
Gebruik dit om de app te builden en direct naar testers te pushen via Firebase:
```bash
./gradlew assembleRelease appDistributionUploadRelease
```

### 3. Database Indexen Updaten
Als er nieuwe query-fouten zijn in de webconsole:
```bash
firebase deploy --only firestore:indexes
```

## 🔍 Database Structuur (Firestore)
- `/routes/{id}`: Bevat route metadata, naam en gereden GPX datapunten.
- `/pois/{id}`: Bevat onkruidlocaties, obstakels en andere gemarkeerde objecten.
- `/tasks/{id}`: Bevat live taken die door de AI of admin zijn aangemaakt (Status: open/closed).

## 🚀 AI Live Acties
De AI kan de volgende taken uitvoeren op aanvraag:
1. **"AI, zet de meldkamer online"** -> Voert hosting deploy uit.
2. **"AI, stuur een nieuwe testversie"** -> Voert App Distribution upload uit.
3. **"AI, maak een nieuwe taak aan in de cloud"** -> Gebruikt de Firebase Admin SDK of CLI om een document in `/tasks` te schrijven.
4. **"AI, controleer de live status"** -> Inspecteert Firestore logs voor actieve routes.

*Gereed voor deployment. AI heeft volledige lees/schrijf toegang tot build scripts en assets.*
